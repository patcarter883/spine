# Specification
## Overview
Add a `--verbose` / `-v` flag to the SPINE CLI's top-level Click group (`main`) that toggles Python debug logging across the entire application. Currently, 47 modules declare `logger = logging.getLogger(__name__)` but no `logging.basicConfig()` is ever called during normal CLI execution, leaving all loggers at the default WARNING level. The new flag will configure the Python logging system at CLI startup, mapping `-v` to INFO level and `-vv` to DEBUG level, making the existing but dormant log statements visible to users. A `SPINE_VERBOSE` environment variable will follow the established `SPINE_*` env var convention, allowing verbosity control without CLI flags.

## Requirements
1. Add a `--verbose` / `-v` count option to the `main` Click group in `spine/cli/__init__.py` that accepts multiple occurrences (e.g., `-v` for INFO, `-vv` for DEBUG).
2. Create a new `spine/log.py` module with a `configure_logging(verbosity: int)` function that calls `logging.basicConfig()` with the appropriate level based on the verbosity count (0 → WARNING, 1 → INFO, 2+ → DEBUG) and a consistent log format including timestamp, level, and module name.
3. The CLI `main` group callback must call `configure_logging(verbosity)` early, before any subcommand logic executes, so that log output is visible from the start of execution.
4. Set the `SPINE_VERBOSE` environment variable from the CLI flag value (matching the `--debug-llm` → `SPINE_DEBUG_LLM` pattern) so that child processes (e.g., `spine ui` spawning Streamlit) and downstream code can detect the verbosity level.
5. `configure_logging(verbosity)` must also read `SPINE_VERBOSE` env var as a fallback when the CLI flag is not explicitly set, enabling verbosity control via environment variable alone.
6. The `spine` namespace logger (`logging.getLogger("spine")`) must be configured rather than the root logger, to avoid changing log levels for third-party libraries (e.g., websockets, httpx, urllib3) that would produce excessive noise at DEBUG level.
7. The log format must include `%(asctime)s %(levelname)s %(name)s: %(message)s` for consistency and debuggability.
8. Existing `--debug-llm` flag behavior must remain unchanged and orthogonal to `--verbose` — they control different output mechanisms (LLM callback vs Python logging).
9. Unit tests must verify that `configure_logging(0)` leaves loggers at WARNING, `configure_logging(1)` sets spine loggers to INFO, and `configure_logging(2)` sets them to DEBUG.
10. CLI integration test must verify that `spine run -v` and `spine run -vv` produce log output, while `spine run` (no flag) does not.

## Architecture
- **New module `spine/log.py`**: Centralizes logging configuration in a single importable function `configure_logging(verbosity: int)`. This is the missing piece — currently no module sets up Python logging, so all 47 module-level loggers silently discard INFO/DEBUG messages. The function configures only the `spine` namespace logger with a `StreamHandler` writing to stderr, avoiding interference with third-party library log levels.
  - Rationale: A dedicated module keeps logging setup decoupled from CLI details and reusable by non-CLI entry points (e.g., `python -m spine`, test fixtures, Streamlit app).

- **Group-level Click option on `main`**: Add `@click.option("--verbose", "-v", count=True, default=0, help="Increase log verbosity (-v for INFO, -vv for DEBUG)")` to the `main` group. The group's callback (currently a `pass` no-op) becomes the place to call `configure_logging()`.
  - Rationale: A group-level option makes `--verbose` available to all subcommands without repeating the option on each one. This is structurally cleaner than per-command flags and follows Click best practices for global options.

- **Env var `SPINE_VERBOSE`**: Set in the CLI callback and read as fallback in `configure_logging()`. Maps to an integer (0, 1, 2) following the existing `SPINE_*` convention.
  - Rationale: Mirrors the `--debug-llm` → `SPINE_DEBUG_LLM` pattern. Enables verbosity in child processes (Streamlit subprocess, worker processes) that don't inherit Click context.

- **Namespace-scoped logger configuration**: Configure `logging.getLogger("spine")` instead of the root logger. Third-party libraries (websockets, httpx, urllib3, etc.) stay at their default WARNING level to avoid flooding output.
  - Rationale: A `--verbose` flag that turns on DEBUG for every library would be unusable due to noise. The `spine` namespace covers all `spine.*` loggers created by the 47 existing modules.

- **Callback execution order**: The `main` group callback runs before any subcommand, ensuring logging is configured before `SpineConfig.load()`, `submit_work()`, or any other logged operation begins.
  - Rationale: Early configuration captures startup-phase log messages (config loading, workspace root detection) that would otherwise be lost.

## Interfaces
- **`spine.log.configure_logging(verbosity: int) -> None`**
  - `verbosity=0`: Sets `logging.getLogger("spine").setLevel(logging.WARNING)` with a stderr `StreamHandler`
  - `verbosity=1`: Sets level to `logging.INFO`
  - `verbosity=2+`: Sets level to `logging.DEBUG`
  - Reads `SPINE_VERBOSE` env var as fallback if `verbosity=0` and env var is set (parses as int, capped at 2)
  - Format: `%(asctime)s %(levelname)s %(name)s: %(message)s`
  - Idempotent: safe to call multiple times (checks for existing handlers)

- **`spine.log.get_verbosity_from_env() -> int`**
  - Reads `os.getenv("SPINE_VERBOSE", "0")`, parses to int, clamps to [0, 2]
  - Returns 0 if env var is missing or unparseable

- **Click group option on `spine.cli.main`**
  ```python
  @click.group()
  @click.version_option(version=__version__)
  @click.option("--verbose", "-v", count=True, default=0,
                help="Increase log verbosity (-v for INFO, -vv for DEBUG)")
  @click.pass_context
  def main(ctx, verbose):
      """SPINE - AI agent harness"""
      os.environ["SPINE_VERBOSE"] = str(verbose)
      configure_logging(verbose)
      ctx.ensure_object(dict)
      ctx.obj["verbose"] = verbose
  ```

- **No changes to `SpineConfig`**: The verbose flag operates at the CLI/logging layer, not the config layer. This mirrors how `--debug-llm` bypasses SpineConfig and keeps the config dataclass focused on agent configuration rather than runtime observability.

- **No changes to subcommand signatures**: Subcommands that need to know verbosity can read `ctx.obj["verbose"]` or check `SPINE_VERBOSE` env var. No subcommand parameter changes required.

## Success Criteria
1. Running `spine run -v` produces INFO-level log output from `spine.*` loggers to stderr; running `spine run` (no flag) produces no INFO/DEBUG log output (only WARNING and above).
2. Running `spine run -vv` produces DEBUG-level log output from `spine.*` loggers; third-party library loggers (e.g., `websockets`, `httpx`, `urllib3`) remain at WARNING level and do not emit DEBUG noise.
3. Setting `SPINE_VERBOSE=2` in the environment and running `spine run` (no CLI flag) produces DEBUG-level log output, confirming the env var fallback works.
4. All existing `--debug-llm` functionality continues to work unchanged when combined with or without `--verbose` flags.
5. Unit tests in `tests/unit/test_logging.py` verify `configure_logging(0/1/2)` sets the correct levels on the `spine` namespace logger and does not affect the root logger or third-party loggers.

## Architecture Overview
## Architecture Overview

The SPINE CLI currently uses a flat Click group structure where `main()` is a bare `@click.group()` with no options or callbacks. Debug logging is implemented via `--debug-llm` flag on individual subcommands (`run` and `ui`), which sets the `SPINE_DEBUG_LLM` environment variable and calls `install_global()` to enable LangChain's debug/verbose globals.

The new `--verbose` flag will be added to the top-level `main` group, making it available to all subcommands. When enabled, it will call `configure_logging()` to set up Python's stdlib logging with a consistent format and INFO level.

### Key Changes:
1. **Add `--verbose` flag to `main` group** - Uses Click's `invoke_without_command=True` pattern to configure logging before any subcommand runs
2. **Implement `configure_logging()`** - Centralized logging setup using `logging.basicConfig()` with consistent format
3. **Update subcommands** - Remove duplicate `SpineConfig.load()` calls by leveraging context object pattern
4. **Add tests** - Unit tests for `configure_logging()` and CLI integration tests

### Data Flow:
```
spine --verbose run mytask.yaml
    ↓
main() group callback runs first
    ↓
configure_logging() sets up logging
    ↓
ctx.obj = SpineConfig.load()
    ↓
run subcommand executes with logging configured
```
